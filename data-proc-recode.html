Recode (many) variables

Author

Sven Rieger

Last update on

March 6, 2023

Recoding variables is … For example, psychological measures often contain so-called reversed formulated items. Moreover, sometimes it is necessary to condense categories.

If you need example data, here is some simulated data.

The data generating process follows 3-dimensional confirmatory factor analyses model with categorical indicators. Each latent variable (\(\eta_1,\eta_2, \eta_3\)) is measured by 4 ordered categorical indicators. The last items (i.e., \(Y_4\), \(Z_4\), and \(W_4\)) should be recoded (indicated by a negative factorloading).

For data generation, we used the simulateData function of the lavaan package (Rosseel et al., 2022). If you are interested in simulating data, please see here.

PopMod3dim <- "

eta1 =~ .8*Y1 + .8*Y2 + .8*Y3 + -.8*Y4
eta2 =~ .8*Z1 + .8*Z2 + .8*Z3 + -.8*Z4
eta3 =~ .8*W1 + .8*W2 + .8*W3 + -.8*W4

Y1 | -1*t1 + 0*t2 + 1*t3
Y2 | -1*t1 + 0*t2 + 1*t3
Y3 | -1*t1 + 0*t2 + 1*t3
Y4 | 1*t1 + 0*t2 + -1*t3

Z1 | -1*t1 + 0*t2 + 1*t3
Z2 | -1*t1 + 0*t2 + 1*t3
Z3 | -1*t1 + 0*t2 + 1*t3
Z4 | 1*t1 + 0*t2 + -1*t3

W1 | -1*t1 + 0*t2 + 1*t3
W2 | -1*t1 + 0*t2 + 1*t3
W3 | -1*t1 + 0*t2 + 1*t3
W4 | 1*t1 + 0*t2 + -1*t3

eta1 ~~ 1*eta1 + .3*eta2 + .3*eta3
eta2 ~~ 1*eta2 + .3*eta3
eta3 ~~ 1*eta3
"
dat3dim <- lavaan::simulateData(PopMod3dim,
                                sample.nobs = 1000,
                                seed = 999)

Recode one variable

Before recoding the variables, we should check them (e.g., how many categories?). This can be done using e.g., the table function (when the variables are ordinal/ordered categorical variables).

with(dat3dim, table(Y4))
Y4
  1   2   3   4 
150 370 321 159

In R there are many different options to recode variables. Below are two examples.

  • Approach 1: recode (car)
  • Approach 2: Number of categories

In this approach, we use the recode function of the car package (Fox et al., 2022). This function needs at least 2 inputs (copied from the package description):

  • var: numeric vector, character vector, or factor.
  • recodes: character string of recode specifications

There are further additional arguments such as as.factor and as.numeric which direct the class of the output.

The code is as follows:

dat3dim$Y4r1 <- car::recode(var = dat3dim$Y4,
                            recodes = "
                            1 = 4;
                            2 = 3;
                            3 = 2;
                            4 = 1;",
                            as.factor = FALSE)

A quick double check.

with(dat3dim, table(Y4, Y4r1))
   Y4r1
Y4    1   2   3   4
  1   0   0   0 150
  2   0   0 370   0
  3   0 321   0   0
  4 159   0   0   0
Warning

Note that this approach is not very robust across different recoding strategies. However, for recoding lickert items that are coded like in this example, it is a convenient approach.

In this approach, we subtract the variable \(Y_4\) from the number of categories + 1 (here there are 4 categories 1 to 4).

dat3dim$Y4r2 <- 5 - dat3dim$Y4

A quick double check.

with(dat3dim, table(Y4, Y4r2))

  1   0   0   0 150
  2   0   0 370   0
  3   0 321   0   0
  4 159   0   0   0

Recode many variables

If there are more variables that need to be recoded, you should use loops (see here). For this purpose, it is useful to work with character vector and lists (see here).

The procedure encompasses 3 steps:

  1. Generate a character vector (here: toRec) of items that must be recoded.
toRec <- c("Y4", "Z4", "W4")
  1. In addition, we generate a character vector (here: recoded) that contains the item names of the recoded variables. To concatenated all item names with “r”, we use the paste0 function.
recoded <- paste0(toRec, "r")
recoded
[1] "Y4r" "Z4r" "W4r"
  1. Now, we use a loop (sapply) to iteratively recode all variables.
dat3dim[,recoded] <- sapply(dat3dim[,toRec],
                            function(x) 
                              car::recode(var = x,
                                          recodes = "
                                          1 = 4;
                                          2 = 3;
                                          3 = 2;
                                          4 = 1;",
                                          as.factor = FALSE)
                            )
